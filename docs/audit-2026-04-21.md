# Selah Audit — 2026-04-21

Six parallel Explore agents, each scoped to a different subsystem. Total: **113 findings** (7 Critical, 27 High, 40 Medium, 40 Low). Agents explicitly excluded fixes already shipped in v0.3.1→v0.3.3 (PIN loop, cookie Secure flag, companion-safety UI bugs, logout).

## Top themes (cross-cutting)

1. **Raw error messages leak to clients across the whole AI surface.** Provider adapters, SSE streams, export routes, and crypto decrypt all hand `err.message` straight to the user. At least one path can echo an API key. Needs a `sanitizeProviderError()` helper used everywhere.
2. **Ownership checks on user-local Prisma reads are inconsistent.** Several places filter by a PK without a `userId` check (`thread-store.getThreadMessages`, `appendMessage`), or risk regressing to that pattern (`conversations/[id]` DELETE).
3. **`/api/profiles/[id]` PATCH/DELETE has no caller authentication.** Any browser session — even no session at all — can modify or delete any profile because middleware allow-lists the whole `/api/profiles` prefix.
4. **Seed-merge + disk operations can silently corrupt the DB.** Orphan FKs copied without cleanup, non-atomic rename swap, no free-space check before xz decompression. On a failure mid-swap, user data is gone with no automatic rollback.
5. **Claude 4.x sampling-parameter footgun is not defended.** Direct Anthropic adapter still sends `temperature`/`top_p` under some code paths; per the 2026 API changes, these now *error*.
6. **Write-ahead log in MCP lacks fsync** and has a confusing `status='failed' + error='pending'` pre-flight record that's never cleaned up if the next step crashes.

## Critical (7)

Numbered for triage — fix these first.

1. **[release] `scripts/ops/apply-seed-update.ts:71`** — `mergeUserData()` reports orphan FKs post-transaction but doesn't act on them. Dangling references ship into the new seed and surface as constraint failures on first write. **Fix:** throw if `report.orphanRows.length > 0`, or purge them inside the transaction.
2. **[release] `scripts/ops/apply-seed-update.ts:101-102`** — two sequential `renameSync` calls with no rollback. If the second rename fails, the live DB is gone and `data/selah.incoming.db` must be renamed by hand. **Fix:** wrap in try/catch and restore from backup on failure; ideally use a single atomic swap via temp path.
3. **[ai] `src/lib/ai/providers/anthropic.ts:19-24`** + `src/app/api/ai/send/route.ts:88-92` — direct Anthropic adapter can still receive hard-coded `temperature: 0.7, topP: 0.85` under the `custom` provider path. Claude 4.x errors on non-default sampling params. **Fix:** strip sampling params in the Anthropic adapter itself (belt-and-braces) and stop the `custom` override from passing them through when the backing provider is Anthropic.
4. **[ai] `src/app/api/ai/send/route.ts:107`** — raw `err.message` from provider bubbles to the client. At least one test-connection path can leak an API key in an error. **Fix:** introduce `sanitizeProviderError()` and funnel every provider error through it.
5. **[ai] `src/app/api/ai/conversations/[id]/route.ts:19-26`** — DELETE is currently safe (filters by `{id, userId}` in `deleteMany`) but the pattern is fragile; a refactor to `findUnique` by PK would silently break it. **Fix:** explicit `findFirst({where: {id, userId}})` + `if (!owned) return 401` before delete, mirroring the convention we want in every `[id]` route.
6. **[mcp] `mcp/tools/commentaryWrites.ts:60-61`** — `WHERE (verse IS ? OR verse = ?)` is logically broken for NULL matching; the chapter-level (verse = NULL) upsert path doesn't match rows it intends to update, causing duplicates. **Fix:** `WHERE source_id = ? AND book_id = ? AND chapter = ? AND COALESCE(verse, -1) = COALESCE(?, -1)`.
7. **[ux] `src/lib/crypto.ts:11`** — encryption key derived from `DATABASE_URL` string hash. Low entropy + co-located with the data it's supposed to protect. **Fix:** dedicated `ENCRYPTION_SECRET` env var, generated and persisted at deploy time; fail startup if missing.

**Release/seed/Docker (6)**
- `scripts/ops/apply-seed-update.ts:64` — xz decompression has no free-space preflight. Fails mid-stream on small volumes and corrupts. Add `statSync` check requiring ≥1.5× `sizeRaw`.
- `src/lib/seed/merge-engine.ts:87-92` — removed-column violation is logged but the transaction continues into the INSERT with mismatched column lists. Add `continue` / `throw` at the violation site.
- `docker-entrypoint.sh:26-27` — `check-seed-update` failure swallowed; app boots on partially-migrated DB. Distinguish retryable (exit 2) vs fatal (exit 1) and refuse to boot on fatal.
- `scripts/release/prepare-seed-db.ts:155-161` — preflight violations cause `exit(1)` with no documented recovery path. Add a "preflight failed" section to `docs/release-process.md`.
- `scripts/ops/check-seed-update.ts:77-90` — auto-apply failure logs but leaves no marker for health checks. Write `.seed-update-failed` on fatal errors.
- `src/lib/seed/merge-engine.ts:103` — in-place `DELETE` + `INSERT SELECT` leaves table empty during the window; if INSERT fails on a post-merge CHECK, the rollback protects correctness but the approach is fragile. Swap via a temp table.

**MCP (3)**
- `mcp/lib/writeLog.ts:30` — `appendFileSync()` has no `fsync`. Add `fs.fsyncSync(fd)` after append so the log really is the authority for "did this happen."
- `mcp/lib/writeLog.ts:81-87` — pre-flight log line writes `status='failed' + error='pending'`; if the process dies before the commit line writes, the log has a dangling record that claims it failed. Use `status='pending'` until commit.
- `mcp/tools/themes.ts:42` — `TRIM(t.${col})` interpolates a column name into SQL. `col` is caller-chosen; constrain to a whitelist enum.

## Low (40)

Grouped; fix opportunistically.

- **Validation discipline**: POST endpoints for journals, notes, collections, profiles, study projects all trim inconsistently and lack explicit max-lengths. Build a shared `validateUserString({min, max, trim, normalize})` and use everywhere.
- **Accessibility polish**: focus restoration on `ConfirmDialog`, `not-found.tsx` coverage, theme-switcher active indicator, empty-state copy for character search and journal picker.
- **Mobile**: `maximumScale=5` in viewport; consider `=1` to prevent accidental zoom traps. Safe-area insets not audited.
- **UX conveniences**: no Ctrl+G "go to passage" hotkey, no print stylesheet, docx export UI may be unwired despite the dep being pulled in.
- **Release hygiene**: no rotation on `/app/backups`, `SELAH_AUTO_UPDATE_SEED=1` default is silent (log it), `compareSeedVersions` lexicographic sort relies on YYYY.MM.DD zero-padding not being enforced by a type, CI version extraction not semver-validated.
- **MCP polish**: narrative-unit writes are absent (intentional?), various low-severity error-message tightenings, devotional weakness formula undocumented.
- **AI polish**: `testConnection` uses `max_tokens=1`; some models balk.
