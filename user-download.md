## Step 2: Run Claude Pipeline (Layer 2 Content)

Layer 2 is the pre-baked contextual knowledge that makes Selah more than a plain Bible reader. This content is generated by Claude using the prompts in `selah-prebake-prompts.md`.

### What the pipeline generates

| Content | Table | Description |
|---|---|---|
| Characters | `characters` | ~125+ biblical people with bios, modern parallels, emotional arcs |
| Character appearances | `character_appearances` | Scene cast for each narrative unit |
| Character relationships | `character_relationships` | Family, allies, adversaries connections |
| Themes | `themes` | Conceptual threads with modern framing and scholarly definitions |
| Passage-theme links | `passage_themes` | Which themes surface in which passages |
| Climate contexts | `climate_contexts` | Historical/political/social environment for eras |
| Narrative units | `narrative_units` | Pericopes with three-lens annotations |
| Devotionals | `devotionals` | Daily Bread entries with conversation starters |
| Devotional tags | `devotional_tags` | Life-situation mood taxonomy |

### How to run it

The pipeline uses Claude (or any AI provider) to generate structured JSON for each content type. The prompts are in `selah-prebake-prompts.md` at the project root (`C:\dev\selah-dev\selah-prebake-prompts.md`).

**Option A: Use Claude Code (recommended)**

1. Open a new Claude Code session in the `selah/` directory
2. Tell Claude:
   ```
   Read selah-prebake-prompts.md. Generate Layer 2 content for the Selah database.
   Start with characters — generate the top 125 biblical figures with full profiles,
   then insert them into the characters table using better-sqlite3.
   Follow the prompt templates in the file for each content type.
   ```
3. Claude will read the prompts, generate content batch by batch, and insert directly into the SQLite database
4. Repeat for each content type: characters → themes → climate contexts → narrative units → devotionals

**Option B: Use the Anthropic API directly**

1. Set your API key: `export ANTHROPIC_API_KEY=sk-ant-...`
2. Create a script that:
   - Reads a prompt template from `selah-prebake-prompts.md`
   - Sends it to Claude with structured output (JSON mode)
   - Parses the response and inserts into the database
3. Run for each content type in dependency order:
   - Characters first (other content references them)
   - Themes second (passages reference themes)
   - Climate contexts (referenced by narrative units)
   - Narrative units (the three-lens annotations)
   - Devotionals last (references narrative units and characters)

**Option C: Manual seeding**

For a quick start without running the full pipeline, you can seed a small set manually:
```bash

npx tsx scripts/etl/seed-sample-characters.ts
```

## Step 3: Wire UI to Database (Phase 2)

After data is populated, the next development phase replaces sample JSON in each section with real API routes querying the database. This is done section by section:

1. Create Next.js API routes under `src/app/api/`
2. Replace `sample-data.json` imports with `fetch()` calls to the API
3. Test each section with real data

This step is done by Claude Code — tell it to implement Phase 2 for a specific section.

## Quick Reference: Running Scripts

All commands from the `selah/` directory:

```bash
# Verify current state
npx tsx scripts/etl/07-verify.ts

# Re-run any import (safe — uses INSERT OR IGNORE)
npx tsx scripts/etl/06-populate-books.ts
npx tsx scripts/etl/01-import-translations.ts
npx tsx scripts/etl/02-import-strongs.ts
npx tsx scripts/etl/03-import-crossrefs.ts
npx tsx scripts/etl/04-import-commentaries.ts --fetch
npx tsx scripts/etl/05-import-tyndale-profiles.ts

# Reset database (delete and recreate)
rm data/selah.db
npx prisma db push
# Then re-run imports
```
